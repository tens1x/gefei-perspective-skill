# SEO 实战

> 蒸馏自 哥飞 839 条即刻动态 + 8 篇长文（02-seo）

## Mental models

- **SEO的核心是满足用户的真实搜索需求，谷歌会根据用户行为（停留时间、跳出率）来判断网页质量，能长期拿到流量的一定不是垃圾站**
  - 出处：post_01556, post_05995, post_04721, post_06968
  - 原话：> 假如一个排在搜索引擎第一的网页，搜索者进去后又立马点击其它结果，就会被搜索引擎认为这个网页无法满足搜索者的需求，那么自然排名就会下降。 而一直雄霸，说明提供的服务足够好。
- **做SEO要找新词和低竞争词，新词没有竞争，只要被收录就能排名第一；通过大量长尾小词积少成多获取可观流量**
  - 出处：post_04882, post_06421, post_07296, post_05150, post_07504
  - 原话：> 谷歌每天处理的搜索中有15%是从来没有被搜索过的词。 2016年，谷歌说每年处理2万亿次搜索，到2023年，应该要有3万亿次以上了。 15%就是4500亿，再除以365得到每天是12亿次左右。
- **网站是可以做成一生的事业，像种树一样不断积累，做时间的朋友，养网站防老**
  - 出处：post_06626, post_05531, post_04618
  - 原话：> 做网站就像是种树一样，先种下第一棵树，再种下第二棵树，慢慢你就有了一个小果园，收成会越来越好
- **通过程序化生成大量有价值的页面给搜索引擎收录是SEO的引爆点，每个页面对应一个关键词，积少成多聚沙成塔**
  - 出处：post_04716, post_04725, post_04730, post_05382
  - 原话：> 一个页面可能一天只能带来10个访问，但如果你有1万个这样的页面，是不是每天就有10万个访问了。
- **SEO八个字概括：内修内容，外扩外链。内容为王，外链为皇**
  - 出处：post_05538, post_05539
  - 原话：> 做SEO很简单，八个字就可以概括： 内修内容，外扩外链
- **域名在搜索引擎的权重是最有价值的资产，有了高权重域名，随便什么内容都能排到搜索结果前面**
  - 出处：post_01508, post_04455
  - 原话：> 知乎公司目前最值钱的就是zhihu.com这个域名了。 这个域名在各个搜索引擎里的权重都很高，随便什么内容都能排到搜索结果第一页。
- **SEO不是低级活，SEO高手都是产品洞察高手，需要分析用户需求，做出满足搜索需求的产品页面**
  - 出处：post_05414, post_04538
  - 原话：> 不要一听到SEO三个字就觉得很Low，其实SEO高手都是产品洞察高手，只有懂技术的产品经理才能胜任SEOer的工作。 做SEO需要分析用户需求，做出能够满足用户搜索需求的产品页面。
- **工具类网站一旦做好放上去，搜索引擎会源源不断带来用户，几乎不需要更新内容就能持续赚钱**
  - 出处：post_00679, post_05531
  - 原话：> 以前做工具类网站也是类似，因为工具类不需要更新内容，但用户却能源源不断的从搜索引擎过来，所以每个月的工作就是按时去收美金再结汇成人民币。
- **先从搜索引擎获取小流量开始，不断积累经验，先爬后走再跑，赚钱靠经验积累，先从第一块钱赚起**
  - 出处：post_06198, post_05801, post_06005
  - 原话：> 先爬后走再跑，不要在还没有学会走路时就去考虑怎么跑得快。 先爬一爬，然后站起来走一段时间，再跑起来。
- **AI+工具+出海是当下中小开发者的财富密码，AI自带传播属性，工具满足刚需，出海赚高单价**
  - 出处：post_06009, post_06206
  - 原话：> 记住下面三个词六个字：  AI+工具+出海  这就是财富密码的大方向

## Heuristics (when → then)

- 当 **新网站上线想快速被谷歌收录** ，则 **去V2EX发帖介绍自己产品并留下网站链接，V站在谷歌权重很高，1小时内就可能引来爬虫**
  - 来源：post_04661
  - 原话：> 新上线的网站，如何快速让谷歌收录？ 答案是去V2EX发帖介绍自己产品，然后留下网站链接。  这个方法10年前就有用，现在更有用了，因为V站在谷歌的权重更高了。
- 当 **想学SEO** ，则 **最好的教程是搜索引擎官方出的文档，如谷歌的SEO文档**
  - 来源：post_04719
  - 原话：> 如果你想学习SEO，那么最好的教程就是搜索引擎官方出的文档，如谷歌的SEO文档。 https://developers.google.com/search/docs?hl=zh-cn 你做的网站，能不能从搜索引擎拿到流量，就看你学这个文档学会了多少。
- 当 **发现一个搜索量大但竞争度低的关键词** ，则 **要先看看谷歌搜索结果是否已经直接满足了用户需求，如果是则先避开**
  - 来源：post_06202
  - 原话：> 有时候你找到一个词，一看搜索量挺大，竞争度很低，很兴奋的就想去做。 这个时候要注意去看看，是不是谷歌的搜索结果下面直接就可以满足用户需求了。 像截图这种情况，就可以先避开。
- 当 **已有流量的网站改版时** ，则 **新版需要与旧版的url规划设计一致，保持URL不变**
  - 来源：zhihu_p646875640
  - 原话：> 对于已经有流量，已经被搜索引擎收录的网站，改版时，新版需要与旧版的 url 规划设计一致。
- 当 **做好一个有流量的网站之后** ，则 **围绕同一个关键字多做几个网站，做矩阵，老带新，靠老网站给新网站增加外链和访问量**
  - 来源：post_06626
  - 原话：> 老带新，只要做出来了一个有一定流量的网站，就可以不断地做新网站，做矩阵。靠老网站给新网站增加外链和访问量，来提升新网站的权重和排名
- 当 **想做SEO但不知道做什么需求** ，则 **去similarweb查看竞品域名数据，通过一个域名发现10个以上新域名，不断深挖，只看简单的能快速实现的**
  - 来源：post_04648
  - 原话：> 现在我给你一个有效办法，可以去 https://similarweb.com ，先随便扔进去一个你熟悉的网站的域名开始，去查看数据，去看热门受众兴趣点和竞对网站，通过一个域名你能找到10个以上的新域名
- 当 **判断一个搜索关键词是否值得做** ，则 **看谷歌搜索结果前十是否都是首页（竞争大），如果很多是内页则值得做**
  - 来源：podcast_ep57
  - 原话：> 如果一个搜索关键词，谷歌搜索结果前十都是首页，那么说明竞争很大，如果很多都是内页，那么就值得做
- 当 **网站想从谷歌获得流量** ，则 **请一定后端渲染，不要前端渲染，谷歌不会为小网站花算力执行JS**
  - 来源：post_07478
  - 原话：> 谷歌默认抓取页面时，只抓取 html 的 body 内部内容，那么放在 script 标签里的 json 当然就不会被抓取了。  当然你会说，谷歌会去执行 js 代码。但是执行js是需要多花费算力的，谷歌凭什么给你的新网站这种待遇呢？  所以，各位请注意，如果你的网站想要从谷歌获得流量，请一定后端渲染。
- 当 **做SEO起效最快** ，则 **找新词，新词竞争少甚至没有竞争，最快第二天就能从谷歌获取流量**
  - 来源：post_06421
  - 原话：> 真的有效 //@赖嘉伟Gary:  今天做 SEO 起效最快的方法是——找新词。
- 当 **网站排名被竞争者超过** ，则 **保住排名的方法很简单，比竞争者做得更好**
  - 来源：post_05511
  - 原话：> 排名会掉是因为有竞争者，而保住排名的方法也很简单，比竞争者做得更好
- 当 **拿到排名的时候** ，则 **比现在排在前面10个位置的网页做得更好，你只需要跟10个网页竞争**
  - 来源：post_06748
  - 原话：> 其实拿到排名的秘诀一句话就可以概括： 比现在排在前面10个位置的网页做得更好。  你不需要跟全世界竞争，你只需要跟10个网页竞争。
- 当 **做一个网站首页** ，则 **一定要有一个H1标签，让搜索引擎知道网站的核心关键字是什么；每个页面都要让搜索引擎能够轻松抓取到主题**
  - 来源：post_00040
  - 原话：> 首页一定要有一个H1标签，让搜索引擎知道网站的核心关键字是什么
- 当 **产品开发完成后** ，则 **开发完成不是项目结束而是推广的开始，要花几倍于开发的时间来推广产品**
  - 来源：post_05588
  - 原话：> 其实我们做产品也应该是这样的，产品开发完成不是项目的结束，而应该只是推广的开始。 之后要花几倍于开发的时间来推广你的产品，用尽各种免费的方法，然后尝试付费的渠道。
- 当 **给网站增加多语言支持后** ，则 **访问量会显著增加，甚至可以增加10倍**
  - 来源：post_06677
  - 原话：> @PowerLin 说，他给博客增加多语言支持后，访问量增加了10倍。
- 当 **想通过SEO获取小众需求的流量** ，则 **做一个网站让搜索引擎收录并给排名，即使全球每天只有100个人有这个需求也能精准导入**
  - 来源：post_06049
  - 原话：> 为什么要做SEO？ 一个很好的解释是，一些比较小的需求，或者比较低频的需求，你想主动去找用户，会很困难。 但如果你做一个网站，让搜索引擎收录你并且给你排名，那么即使全球每天只有100个人会有这个需求，也能够精准的导入到你的网站里。

## Signature phrases

- "养网站防老" — 将做网站比喻为种树/养老，强调网站是可以长期持有的资产，类似被动收入 （post_06626, post_06864, post_06970）
- "内修内容，外扩外链" — 对SEO核心方法论的八字概括，内容为王外链为皇的精简表达 （post_05538, post_05539, post_06957）
- "AI+工具+出海" — 给中小开发者的财富密码大方向，三个关键词合并的核心策略 （post_06009, post_06206, post_05027）
- "找新词" — SEO起效最快的方法，新词竞争度低容易排到第一，是反复强调的核心策略 （post_04882, post_06421, post_07296, post_05150, post_05167）
- "先爬后走再跑" — 劝告初学者不要急于求成，先做出网站获取流量再说，循序渐进积累经验 （post_06198, post_06259, post_06260）

## Anti-patterns

- **不要前端渲染网页内容，要后端渲染** — 谷歌默认只抓取HTML的body内容，不会为小网站花算力执行JS代码，前端渲染导致谷歌无法收录页面内容
  - 来源：post_07478
  - 原话：> 谷歌默认抓取页面时，只抓取 html 的 body 内部内容，那么放在 script 标签里的 json 当然就不会被抓取了。  当然你会说，谷歌会去执行 js 代码。但是执行js是需要多花费算力的，谷歌凭什么给你的新网站这种待遇呢？  所以，各位请注意，如果你的网站想要从谷歌获得流量，请一定后端渲染。
- **不要为了差异化而去差异化** — 当没有经验时根本做不出差异化，用户需求摆在那里已经通过搜索词表达出来了，先做出网站获取流量再说
  - 来源：post_06198
  - 原话：> 别想着差异化。  你想再多，不如先做出一个网站，先获取到流量再说。  因为当我们还没有经验时，根本做不出差异化。
- **不要用AI生成垃圾文章做SEO** — 垃圾文章即使把访客骗进来了，也无法转化为有效顾客，谷歌只看内容质量不看生成方式
  - 来源：post_05361
  - 原话：> 垃圾文章没前途。  因为我们要靠文章吸引潜在顾客来购买服务或者购买产品，就一定不能是垃圾文章。
- **不要多语言用JS切换** — JS切换不会改变网址，导致谷歌爬虫只能抓取一种语言的内容，多语言就白做了
  - 来源：x_1783679227738653102
  - 原话：> 为什么多语言不能用js切换？ 哥飞答：因为js切换不会改变网址，这就导致谷歌爬虫来的时候，只能抓取到一种语言的内容，也就只能参与一种语言的搜索排名。那么这个网页的多语言就相当于白做了。
- **不要在meta keywords里堆砌大量无关关键字** — 搜索引擎已经降低了keywords权重，只写title和description即可
  - 来源：post_00638
  - 原话：> 后来搜索引擎排名算法就降低关键字的权重了，所以我的SEO风格就是只写title和description，不写keywords。
- **不要沉迷于免费流量而忽略放大优势** — 应该在能赚钱的地方深挖下去，不应觉得花一点时间就能做出来而不屑一顾
  - 来源：post_03469
  - 原话：> 我们还是应该放大我们的优势，哪里有流量，哪里能赚钱，就应该在哪里深挖下去。 而不应该觉得好像这是自己花一点时间就能做出来的，没啥大不了的，就不屑一顾，不重视。
- **不要使用子域名放内容，用主域名子目录** — 子目录形式的内容能把权重传递给主域名首页
  - 来源：post_00038
  - 原话：> 不要使用子域名，可以直接用主域名，子目录形式放内容，如 flomoapp.com/weekly/20211117.html 。
- **不要一个产品不成功就彻底放弃** — 应该多做几个，快速试错，快速换方向
  - 来源：post_05027
  - 原话：> 不要一个产品不成功就彻底放弃，多做几个，快速试错，快速换方向
- **不要追求完美产品再上线** — 能跑了就上线，赚到第一块钱再去优化迭代放大
  - 来源：post_05027
  - 原话：> 不要去追求完美产品，能跑了就上线，赚到了第一块钱，再去优化迭代放大
- **不要想清楚再做** — 做了才能想清楚，时间是最大的魔法
  - 来源：post_05027
  - 原话：> 不要想清楚再做，而是做了才想清楚

## Honest limits

- 反向搜索引擎的商业模式（post_03184）：> 话说，我应该（自认为）搞懂这个反向搜索引擎的核心技术了。 也就是说，有钱有时间，我也能搞一个。 但是暂时我没想明白这东西怎么赚钱，以及开发出来需要比较多时间，也需要比较多的钱，所以暂时不会去搞。
- 错过婚纱外贸赚钱机会，缺乏与人交流导致信息差（post_05058）：> 但是我没有深入分析下去，为什么赚钱，以及如何更赚钱。 直到几年后，跟人交流才知道，那几年，他们做婚纱外贸，赚了很多钱。
- 曾经有赚钱的英文站群但没坚持深挖（post_03469）：> 但是，可惜，没坚持。 当时总觉得这东西不是正道，没有花更多精力下去，上了几个站点后就放在那里，没去管它。
