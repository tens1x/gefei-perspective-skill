# 案例拆解与竞品分析

> 蒸馏自 哥飞 201 条即刻动态 + 1 篇长文（09-case-study）

## Mental models

- **内容离钱越近、用户购买欲望越强烈，广告单价就越高——不能简单地说中文站单价低、英文站单价高，要看内容和用户群**
  - 出处：post_00012, post_11846, post_05859
  - 原话：> 不能简单的说，中文网站的点击单价就低，英文站的点击单价就高，还是要看网站的内容和用户群，内容离钱越近，价格就越高，用户群的购买欲望越强烈，价格也越高。
- **放大自己的优势，在自己熟悉的领域做大做强，不要盲目追求别的领域——哪里有流量、哪里能赚钱，就应该在哪里深挖**
  - 出处：post_03469, post_03470
  - 原话：> 我们还是应该放大我们的优势，哪里有流量，哪里能赚钱，就应该在哪里深挖下去。 而不应该觉得好像这是自己花一点时间就能做出来的，没啥大不了的，就不屑一顾，不重视。
- **网站不一定要做多复杂，只需要解决一两个核心需求就行——一个函数、一个功能也可以有人付费使用**
  - 出处：post_06606, post_04905, post_02642
  - 原话：> 看完案例，你就会发现，原来这样也可以，原来网站不一定要做多复杂，只需要解决一两个核心需求就行。
- **地理隔离和语言隔离使得没有谁能完全吃下全球市场，因此后来者总有机会——做多语言、做本地化支付就是切入点**
  - 出处：post_07673, post_08724, post_09180
  - 原话：> 只要放眼全球，就全是机会。 因为有地理隔离和语言隔离存在，没有谁能够完全吃下全球市场，所以我们就有了机会。 别人没做多语言，那我们去做多语言。 别人没做本地化支付，那我们去做本地化支付。
- **赚钱不是线性的，越早开始积累、越持久投入，势能越足——先爬后走再跑，日拱一卒、积少成多**
  - 出处：post_12002, post_12227, post_13530
  - 原话：> 再次证明，只看，只做笔记，不参与，是不会有实际的收获的。而真正开始去做了，才有可能拿到正反馈。举例你说11月做的网站，现在有了日1K的UV，这就是很好的正反馈。 一点一点来，积少成多，聚沙成塔，集腋成裘，收获会越来越大的。
- **在别人能赚钱的红海市场里小赚一点，比去全新蓝海从零开始更容易——不要为了差异化而差异化**
  - 出处：post_09180, post_09960, post_06198
  - 原话：> 彪哥说过类似的话，没有人能够把市场全部吃下，在别人能够赚钱的市场里，你小赚一点还是很容易的。 别人月入100万，你进去月入5万，这可比你找一个全新的没人去做的市场赚5万更容易。
- **做产品不能只开发不推广，花40%时间挖需求、20%开发、40%推广——很多产品失败根因是'无推广运营'**
  - 出处：post_10759, post_13672
  - 原话：> 建议这样安排时间，花40%的时间在挖掘需求上，花20%的时间在开发上，花40%的时间在宣传推广上。
- **精准流量不需要太大量级就能赚钱——一天100UV如果足够精准，也能有两三单付费**
  - 出处：post_10712, post_09653, post_08652
  - 原话：> 谁能想到，一天100UV的网站，也能有一天两三单付费。  只要你的流量够精准，只要你真的能够解决用户需求，就不怕没人给你付费。
- **早就是优势，快就是优势——追热点时比别人更早上线，自带传播性**
  - 出处：post_07192, post_07153
  - 原话：> 群的价值，也许就是比别人早一点看到需求。 早就是优势，快就是优势。
- **做网站是投入回报特别高的生意，最低成本就是域名注册费，但回报可以远超投资百万开店**
  - 出处：post_13825, post_00009
  - 原话：> 就像做 Web 网站，最低成本就是域名注册费，一个域名一年才百来块，谁都出得起这个钱。 但回报却一点也不必那些投资几十万上百万的生意低。
- **拆解目标、量化路径是实现收入目标的关键方法——把月入目标换算成日UV、关键词搜索量、付费率的层层分解**
  - 出处：post_05014, post_11913, post_12024, post_13577
  - 原话：> 很多事情，换个单位，就敢想了。 举例，你觉得月入20万很难，想都不敢想。 但是日入1000刀，月入3万刀，换算成人民币就超过20万了。

## Heuristics (when → then)

- 当 **想估算一个网站的Adsense月收入** ，则 **以30美元为基数（1000日UV×1ECPM×1PV/UV×1广告展示/PV×30天），然后乘以UV倍数、ECPM倍数、PV/UV比、广告展示/PV比**
  - 来源：post_05859
  - 原话：> 所以，记住30美元这个基数，然后乘以其它几个系数就可以快速计算了。
- 当 **网站改版时已有流量和搜索引擎收录** ，则 **新版需要与旧版的URL规划设计保持一致，否则会丢失已有排名**
  - 来源：zhihu_p646875640
  - 原话：> 对于已经有流量，已经被搜索引擎收录的网站，改版时，新版需要与旧版的 url 规划设计一致。
- 当 **想让新网站快速被谷歌收录** ，则 **在V2EX等高权重网站发帖，可以在1小时内吸引谷歌爬虫来抓取**
  - 来源：post_04815
  - 原话：> 就是这个帖子，在1小时就把爬虫给吸引过来了。
- 当 **找到一个正在赚钱的竞品网站** ，则 **不需要差异化，直接做一个满足同样核心需求的产品，把产品做得比竞品更好用、增加免费额度让用户多停留**
  - 来源：post_09960
  - 原话：> 找到别人产品用户愿意买单的点，你自己做一个产品，也满足这个点，就足够了。  然后你要思考的是，如何更好的满足用户，而不是挖空心思想跟竞品的差异化。
- 当 **别人讲的教程方法经验你理解困难** ，则 **分步骤拆解，每一步都带着目的去做某件事情**
  - 来源：post_04741
  - 原话：> 别人讲的教程、方法、经验等，如果你没有相关的积累，的确会理解困难，那么就需要去分步骤拆解了。
- 当 **ChatGPT等新热点出现时** ，则 **围绕热点做工具站，用多种方式变现：卖账号、部署公众号/小程序、做垂直需求产品、做内容站点获取搜索流量**
  - 来源：post_02243
  - 原话：> 做一个垂直需求的产品，基于 ChatGPT 提供的能力，输出内容，卖内容或者卖服务
- 当 **Adsense被风控限制广告展示（因流量增长太快）** ，则 **不能卸载广告代码，要一直放着，让算法持续感受流量质量，一般两到四周恢复**
  - 来源：post_14002
  - 原话：> 不能卸载，就得一直放着，让算法持续感受到你的流量，判断出你的流量质量，之后就能够慢慢恢复广告填充率了。
- 当 **看到小流量网站** ，则 **不能只看流量大小，要看每个UV价值多少钱——很多被忽视的小流量网站正在默默赚钱**
  - 来源：post_13482
  - 原话：> 我们看网站看案例，不能只看流量，不能只盯着大流量网站去看，而要去看对方如何变现的，大概每个 UV 价值多少钱。  其实有很多被我们忽视掉的小流量网站正在默默赚钱。
- 当 **网站有流量但广告收入低** ，则 **检查广告位数量是否足够（谷歌推荐每页最多3个），检查页面类型是否够丰富（标签页、搜索页等增加PV/UV比）**
  - 来源：zhihu_p646875640
  - 原话：> 所有的页面，哥飞都放满3个广告位了，因为这是谷歌Adsense文档里推荐的最大广告位数量。
- 当 **判断一个AI需求是否值得做** ，则 **看两个点：AI的介入是否提升使用体验；这个需求在AI之前是否已有足够的商业价值（用户已经愿意付费）**
  - 来源：post_13871
  - 原话：> 一个需求值不值得做，看两个点。 第一个点是AI的介入是否会提升使用体验。 第二个点是这个需求在之前是否有足够的商业价值。
- 当 **想让网站从谷歌获取更多流量** ，则 **真正满足用户搜索需求，让用户停留时间长、访问多个页面，谷歌就会提升排名给更多流量**
  - 来源：post_07779
  - 原话：> 谷歌对于能够满足用户搜索需求的网站，是不会吝啬的，会很愿意给流量。 因为用户在谷歌能够搜索满足搜索需求，就不会跑去用别的搜索引擎。
- 当 **SEO流量几乎零成本获取时** ，则 **同样金额的MRR利润率远高于靠付费买量的团队，不能只比数字**
  - 来源：post_13276
  - 原话：> 我们这伙人，流量主要来源是SEO，几乎没成本，所以利润率高。 而别的团队，可能大部分流量来自于付费买量，几乎没利润。

## Signature phrases

- "积少成多，聚沙成塔" — 用于描述通过长尾关键词、多个小网站、持续积累的方式逐步提升总流量和收入 （post_04730, post_12002, post_12227）
- "养网站防老" — 哥飞的核心理念之一，把网站当作长期事业来经营，持续产生被动收入 （post_07738, post_07835, post_12288）
- "你只管上站，剩下的交给谷歌" — 鼓励大家动手做网站，靠SEO从谷歌获取免费流量 （post_07678, post_07779, post_07113）
- "先爬后走再跑" — 新手不要急于求成，先跑通流程、积累经验，再追求更大目标 （post_06198, post_11087, post_12399）
- "月入万刀" — 哥飞社群中独立开发者的重要里程碑目标，多人实现后分享经验 （post_12807, post_12060, post_13094, post_13530）
- "不浪费每一个PV" — 在每个页面合理布局广告位，充分利用每一次页面访问的变现价值 （post_00764, post_02555, post_04730）

## Anti-patterns

- **不要为了差异化而差异化，不要在还没学会走路时就去考虑怎么跑得快** — 当我们还没有经验时，根本做不出差异化，先做出来获取流量再说
  - 来源：post_06198
  - 原话：> 别想着差异化。  你想再多，不如先做出一个网站，先获取到流量再说。  因为当我们还没有经验时，根本做不出差异化。
- **不要只开发不推广，不要觉得产品开发完了用户就会蜂拥而来** — 真正失败原因是'无推广运营'，花了大量资金开发却不做推广就等于浪费
  - 来源：post_13672
  - 原话：> 真正失败的原因其实他开头提到了： “最终APP开发完成，无力进一步迭代、推广运营”。
- **不要对自己能做出来的东西不屑一顾，觉得没什么大不了** — 在另一些人手里却能发挥大作用，一个简单的demo可能在抓准用户需求后做到日PV十万
  - 来源：post_03470
  - 原话：> 我们有时候总是会对自己能做出来的东西不屑一顾，觉得没什么大不了。 但其实，在另一些人手里，却能发挥大作用，是宝贝。
- **不要只看不做、只做笔记不参与实践** — 只看只做笔记不会有实际收获，真正开始去做了才有可能拿到正反馈
  - 来源：post_12002
  - 原话：> 再次证明，只看，只做笔记，不参与，是不会有实际的收获的。而真正开始去做了，才有可能拿到正反馈。
- **不建议使用子域名做多语言，更建议使用子目录** — 子目录方式对SEO更友好，能积累主域名权重
  - 来源：post_08724
  - 原话：> 再聊多语言：为什么不建议使用子域名而更建议使用子目录
- **不要一上来就做多语言，新手先踏踏实实做好一个语言** — 多语言维护成本翻很多倍，AI翻译的词不一定是用户真正搜索的词，先做好一个语言拿到流量再扩展
  - 来源：post_13871
  - 原话：> 我发现很多新手一上来就喜欢做多语言，整了一堆垃圾海量页面
- **不要拍脑袋想需求，不要随便做一个自己也有需求的产品** — 找需求有方法可循，应该通过关键词搜索量等数据来发现已被验证的需求
  - 来源：post_10422
  - 原话：> 很多程序员第一次做产品，不知道如何找需求，于是就随便做一个自己也有需求的产品。 其实找需求是有方法可循的
- **不要在流量增长太快时把Adsense广告代码卸载掉** — 需要让算法持续感受到你的流量质量，卸载后反而会更难恢复
  - 来源：post_14002
  - 原话：> 不能卸载，就得一直放着，让算法持续感受到你的流量，判断出你的流量质量，之后就能够慢慢恢复广告填充率了。
- **不要觉得web已经过时，盲目认为必须做移动端App** — 尤其AI赛道火热后大家发现web的好处，几乎产品首发都是web形态
  - 来源：post_03470
  - 原话：> 但其实 web 并没有过时，尤其是最近 AI 赛道的火热，大家又发现 web 的好处了，几乎产品首发都是 web 形态。
- **不要不好意思推广自己的网站** — 推广和SEO两个都缺一不可，不推广流量就很小
  - 来源：post_07450
  - 原话：> 实际上，推广和SEO两个都缺一不可，大家对于自己的网站，还是要多多上心，多多推广，不要不好意思推广。

## Honest limits

- 移动端开发能力（post_03470）：> 我前段时间就有点想偏了，总觉得 web 已经过时了，要做产品就要做移动端 App ，但我又不会移动端开发，所以每次都陷入死循环，要做产品就要先学移动端开发，要学移动端开发就要花时间，要花时间就觉得太浪费时间，还是算了吧……
- 高价社群的交付能力（post_12721）：> 但我发现我暂时交付不了10000块钱一年的社群服务，于是经过两三天的思考，我决定给大家退款。
