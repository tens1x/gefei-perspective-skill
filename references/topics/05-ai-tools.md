# AI 工具与效率

> 蒸馏自 哥飞 207 条即刻动态 + 1 篇长文（05-ai-tools）

## Mental models

- **AI 编程工具的核心价值在于降低门槛，让非程序员也能做产品，但懂产品、懂 SEO、懂交互的综合能力才是真正的护城河**
  - 出处：post_10728, post_11421, post_13134, post_10896
  - 原话：> 产品经理再也没有借口不会写代码了，再也不会就缺一个程序员了。
- **Claude 在编程能力上已经领先 GPT，是当前写代码的最佳选择，并且 Claude 的编程能力左右了整个 AI 行业的竞争格局**
  - 出处：post_10524, post_10583, post_10831, post_15250
  - 原话：> 感觉在写代码上 Claude 已经甩 GPT 一大截了。
- **先做出产品占住生态位，随着大模型能力提升产品效果自然会变好——等到完美再出手就会错过窗口期**
  - 出处：post_10792, post_12645, post_03145
  - 原话：> 创业瞄准大需求，先把能够跑完完整流程的产品做出来，即使效果不是那么好也没关系。  随着大模型越来越强，只要你的产品背后大模型一换，效果立马就变好了。
- **AI 工具站是当前最好的出海创业形态：易开发、易传播、易爆发、易变现**
  - 出处：post_07774, post_14281, post_04061
  - 原话：> AI 工具站有几个特点： 1、易开发，只需要套壳对接 API，然后再做个简单的 Web 界面即可； 2、易传播，目前大家普遍对于 AI 产品的热情高，好奇心强，能够形成用户自发传播； 3、易爆发，选对关键词，网站上线一周就达到每日一万访问也不是不可能； 4、易变现，基础收入靠广告，进阶收入靠用户付费，用户付费意识高，市场已经被教育过了。
- **用 AI 辅助带新人比传统方式更高效：给出大方向，让新人自己跟 AI 对话解决问题，导师只负责纠偏**
  - 出处：post_10647, post_11437
  - 原话：> 给他安排了任务时，先大概讲清楚，然后就让他去跟AI对话，搞清楚需求，再指挥AI去写代码。  期间我时不时问问进展，及时纠偏。
- **做 AI 产品的本质是批发各种模型的 Token 重新打包后卖给用户，定价和产品摆放策略决定了收益**
  - 出处：post_14603, post_14895
  - 原话：> 本质我们就是去批发各种模型的 Token，然后再重新打包后卖给用户。 所以怎么打包，怎么定价也是学问。
- **早就是优势，早就能成为标准——先行者在 AI 领域可以设定行业标准和数据格式**
  - 出处：post_04067, post_03145
  - 原话：> 早就是优势，早就是标准。
- **用现成的 API 服务比自己部署开源模型更划算——OpenAI 可能在以成本价或亏本价提供服务**
  - 出处：post_08565, post_05880
  - 原话：> 对比之后会发现还是用 OpenAI 的API更便宜。  这可能的原因是我们单独租用GPU服务器时，平台赚了我们的钱。  而OpenAI为了占领市场，不断降价，很可能在以成本价或者亏本价格给我们提供API服务。  所以我们多用OpenAI的API就是在占便宜。
- **学习方式已经改变——不是自己去学新技术，而是让 AI 工具（如 Cursor）去学习新技术，自己只需要会指挥**
  - 出处：post_12224, post_14852
  - 原话：> 学习方法的变化 以前是：谢谢推荐，我去学一下。 现在是：谢谢推荐，我让 Cursor 学习下。
- **任何工具都不可能教会所有人，能让有心人提效、降低门槛就足够了**
  - 出处：post_11143, post_10896
  - 原话：> 任何工具，任何方法，任何教程，都不可能教会所有人，让有心人能够提效，降低门槛，就足够了。

## Heuristics (when → then)

- 当 **需要快速上线一个 AI 产品时** ，则 **用 Vercel AI 模板中心选模板，一键部署，几乎不需要什么技术就可以搭建起来**
  - 来源：post_05314
  - 原话：> 不会编程，就不能做 AI 相关产品了吗？不用开发，如何 10 分钟上线一个 AI 产品呢？ 今天哥飞给大家推荐 Vercel AI模板中心，这个大宝库。
- 当 **要做出海网站时** ，则 **用你最熟悉的技术栈，不要花时间学新框架，先把 MVP 做出来最重要**
  - 来源：zhihu_p1952345403925192806
  - 原话：> 你如果熟悉 PHP 建站，那你就用 PHP，你如果熟悉 Python 建站，那就用 Python ，总之，用你最熟悉的方式，最快先把 MVP 做出来，而不是去学习你不熟悉的工具和平台，最终浪费掉了大量时间。
- 当 **产品经理想动手做产品但不会编程时** ，则 **直接学 HTML + TailwindCSS + JS + Next.js，每天一两小时加周末8小时，30小时就可以开始做产品**
  - 来源：post_13134
  - 原话：> 个人建议，可以从 HTML 开始，结合 TailwindCSS 实现的一些 UI ，你可以很快做一个不丑甚至还挺漂亮的网页。  之后再学一点 JS，再学 Next.js 框架，不需要学到能够找程序员工作的程度，只需要学会能够读写数据库，能够动态展示网页数据就可以了。
- 当 **需要让 AI 辅助建站时** ，则 **先用 AI 做出 UI 第一版，下载代码放到 Cursor 里继续修改**
  - 来源：post_12804
  - 原话：> 我的建站流程通常是用 AI 先做第一版，可以不做交互和功能，只做 UI ，然后下载代码，放到 Cursor 里继续修改。
- 当 **同时有多个 AI 代码生成工具可用时** ，则 **把同一个任务同时发给 V0、Loveable、Bolt、Same 等多个工具，看谁做得好就用谁的代码**
  - 来源：post_12769
  - 原话：> 现在，你只需要把一个任务同时给 V0.dev、Loveable.dev、Bolt.new、Same.dev ，你可以得到四个不一样的设计。 之后看谁做的好，你再下载下来，放进 Cursor 里继续调整。
- 当 **Cursor 生成了大量代码后出现越改越错的情况时** ，则 **核心逻辑还是要自己写，让 Cursor 打下手会更好**
  - 来源：post_11178
  - 原话：> 同感，核心逻辑还是要自己写，再让 Cursor 打下手，会更好
- 当 **AI 对话中遇到复杂问题回答错误时** ，则 **把问题拆成多个小问题分步骤提问，AI 就能答对**
  - 来源：post_12165
  - 原话：> 这题让Claude直接解答，回答是错误的。但如果拆成两个问题，先问整堆煤最少有几吨，再来回答哪一天更多，就能答对了。
- 当 **需要对接 AI 大模型 API 时** ，则 **初期自己直接对接，不需要上 LangChain 这样的框架**
  - 来源：post_05105
  - 原话：> 建议每个开放者初期都自己对接，没必要上LangChain。
- 当 **想做 AISEO 让 AI 推荐自己的产品时** ，则 **把信息放在有权重的大站里而不是自己的小网站上，AI 会根据搜索结果来总结内容**
  - 来源：post_12512
  - 原话：> 更好的做法是把自己想让AI看到的信息放在一些有权重的大站里。  最好同时留下网址，甚至不需要链接形式，文字URL就可以了，就已经能让AI看懂了。
- 当 **想搭建本地模型还是用现成 API 时** ，则 **别急着搞本地模型，先用现成的 API 服务，本地模型除了省钱全是缺点**
  - 来源：post_15048
  - 原话：> 玩新东西，最好是先别急着全都自己搞定，能用现成的服务就用现成的。 拿模型来说，本地模型的确省钱，但除了省钱这一个优点外，其他全是缺点，配置要求高、部署麻烦、智力不高，而且可能会有各种不可预料的问题等着你解决。
- 当 **需要让 Claude 帮忙生成内容（如教程、文档）时** ，则 **让 Claude 基于你的代码和需求来总结生成，因为 AI 知道你的功能和实现，所以能准确写出教程**
  - 来源：post_10682
  - 原话：> 我的代码就是 Claude 写的，所以里边的教程也是 Claude 基于代码和我提的需求总结出来的。 这种模式挺好的，AI知道你的功能，知道你的实现，所以能够准确的写出教程。
- 当 **发现一个需求但还没有好的 AI 模型效果来支撑时** ，则 **先做产品占住生态位，等到好的模型出现时产品自然就爆发了（如 Cursor 等到 Claude Sonnet 3.5）**
  - 来源：post_12645
  - 原话：> 试想一下，如果 Cursor 等到了 claude sonnet 3.5 发布之后，才去开始动手做，那么机会还会是 Cursor 的吗？
- 当 **代码还在使用旧的 GPT 模型时** ，则 **及时检查并更新为最新模型，旧模型又贵能力又弱**
  - 来源：post_09259
  - 原话：> 一个小提醒，查查你们代码还有没有在使用gpt-4-32k这个模型，这是又老又贵的旧模型，请赶紧改成gpt-4-turbo模型，能力更强，价格更便宜。

## Anti-patterns

- **不要乱填你的 API Key，尤其是有特殊权限的号** — 别人得到你的 API key 后可以盗你的号，OpenAI 账号有漏洞
  - 来源：post_04459
  - 原话：> 所以不要乱填你的 key ，尤其是你有特殊权限，如Plugin 开发者权限、GPT4API 权限的号。 也不要同意别人说借用你的 api key 做测试。
- **不要用 AI 做事时放弃自己擅长的方式，去试用不熟悉的新工具** — 会浪费大量时间，最终还得回到自己擅长的方式
  - 来源：zhihu_p1952345403925192806
  - 原话：> 所以，总结一下，这次写 PPT 浪费时间，主要原因是放弃了自己擅长的做事方式，试图用一个新工具来实现。  这就告诉我们，做事情，还是要先用自己熟悉的擅长的方式去做。
- **不要让 Cursor 全自动驾驶写复杂项目的核心逻辑** — AI 生成大量代码后你不了解代码，遇到小问题容易越改越错，之前正常的逻辑也可能被破坏
  - 来源：post_11178
  - 原话：> 同感，核心逻辑还是要自己写，再让 Cursor 打下手，会更好
- **不要只看 AI 生成的精美总结图片就以为自己理解了** — 大量的实操经验隐藏在细节里，图片无法展示细节
  - 来源：post_12908
  - 原话：> 图片看起来的确很精美，也会有人收藏。 但如果只看图片，是无法了解细节的。 而大量的实操经验，又隐藏在细节里。 所以，你看到别人传播这类看起来精美的图片，就要问一问自己，真的理解了吗？
- **不要因为不完美就不推出产品** — 产品永远有值得改进的点，先占住生态位再不断迭代才是正确策略
  - 来源：post_12645
  - 原话：> 有些人开发产品的心态是，我不打磨到100分，我就不推出来。  然而，其实产品永远有值得改进的点，难到那就一直改进，一直不推出吗？
- **不要利用别人免费开放的接口批量拉数据再转卖** — 会导致接口被限制，大家都没得用，而且无法兑现已收费的承诺
  - 来源：post_14230
  - 原话：> 本来这就是别人故意露出来的一个免费接口，大家自己用用得了。 就有人要去利用这个接口批量拉数据，还把数据包装一下卖给同行。  现在，这个接口开始限制访问了
- **不要为难自己一上手就去学做 App** — 网页特别容易被传播和使用，不需要审核，部署在免费基础设施上成本极低
  - 来源：post_13134
  - 原话：> 你没必要为难自己，一上手就去学App是不明智的，特别轻量级的网页绝对是你的最佳选择。
- **不要相信 AI 说自己能联网的话** — AI 会根据你的问题迎合你来回答，实际并没有联网能力
  - 来源：post_05603
  - 原话：> 我记得看到有即友说 Claude2 可以联网，然后我实际测试发现，根本不能联网，就是胡说八道。 为什么说是胡说八道呢？看图二就知道，他是根据你的问题迎合你来回答的。 所以结论就是，Claude2根本就没联网。
- **不要对不懂的领域随便下结论** — 大部分人只会用老经验老眼光看待新事物，自以为看懂了其实没看懂
  - 来源：post_13934
  - 原话：> 对于不懂的领域，我的做法是，要么去调研搞清楚，要么就不随便下结论。
- **不要急着搞本地部署模型** — 本地模型除了省钱以外全是缺点：配置要求高、部署麻烦、智力不高
  - 来源：post_15048
  - 原话：> 拿模型来说，本地模型的确省钱，但除了省钱这一个优点外，其他全是缺点，配置要求高、部署麻烦、智力不高，而且可能会有各种不可预料的问题等着你解决。

## Honest limits

- Threads 自动发内容的实现（post_04839）：> 不过怎么自动发内容，我没接触过，不知道难度高不高。
